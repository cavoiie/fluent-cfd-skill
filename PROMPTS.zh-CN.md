# 提示词模板

[English](PROMPTS.md) | 简体中文

在让 Codex 使用 `fluent-cfd` skill 规划、检查、运行或诊断 Ansys Fluent / PyFluent CFD 任务时，可以直接套用这些模板。

## 完整工作流模板

```text
请使用 fluent-cfd skill 处理这个 Fluent / PyFluent / CFD 任务。

任务目标：
- [说明我要解决的问题，例如：检查 case 是否收敛 / 设置边界条件 / 选择湍流模型 / 用 PyFluent 跑 50 步]

已有文件：
- Case 文件：[路径或文件名]
- Data 文件：[路径或文件名，如有]
- 网格文件：[路径或文件名，如有]
- 其他资料：[实验数据、截图、残差曲线、日志等]

物理问题：
- 流体：[空气/水/其他]
- 流动类型：[内流/外流/旋转机械/多相/传热/可压缩]
- 稳态或瞬态：[稳态/瞬态/不确定]
- 关键目标量：[压降/阻力/升力/温度/流量/混合效率/其他]
- 已知边界条件：[入口、出口、壁面、热边界等]

希望你执行：
1. 先判断当前信息是否足够，不足时列出最少需要补充的信息。
2. 按 Fluent CFD 工作流检查设置逻辑。
3. 说明求解器、物理模型、湍流模型、边界条件和数值格式是否合理。
4. 不只看残差，还要检查监控量、质量/能量守恒和物理合理性。
5. 如果需要调用 PyFluent MCP，请先说明会启动 Fluent 并可能占用许可证。
6. 执行前先做 smoke run，不要直接长时间迭代。
7. 最后给出结论：可运行 / 数值收敛 / 物理可信 / 不可信，并说明原因。

输出格式：
- 问题摘要
- 关键假设
- 检查结果
- 建议修改
- PyFluent/TUI 操作步骤
- 收敛与验证标准
- 下一步
```

## 日常短模板

```text
请使用 fluent-cfd skill 帮我处理这个 Fluent 任务：[任务描述]。

文件：[case/data/日志/截图路径]
目标量：[压降/力/温度/流量/其他]
物理类型：[稳态/瞬态，可压/不可压，传热/多相/湍流等]
边界条件：[入口、出口、壁面设置]

请按 CFD 工作流检查，不要只看残差；同时检查监控量、守恒和物理合理性。
如果需要启动 Fluent，请先说明会占用许可证，并先做 smoke run。
最后给出是否可信和下一步建议。
```

## PyFluent MCP 自动运行模板

```text
请使用 fluent-cfd skill 和 pyfluent MCP 处理这个 case。

Case 文件：
[填写 .cas 或 .cas.h5 路径]

操作目标：
- 启动 Fluent
- 读取 case
- 检查 session 状态
- 先运行 20 步 smoke run
- 如果稳定，再运行 [N] 步
- 保存 case/data 到：[输出路径]
- 关闭 Fluent session

要求：
- 启动前说明会占用 Fluent 许可证。
- 每一步都说明调用的 MCP 工具。
- 如果出现发散、回流、负体积、许可证失败或 TUI 报错，立即停止并诊断。
- 不要默认启用 execute_python，除非我明确允许。
```

## 收敛诊断模板

```text
请使用 fluent-cfd skill 诊断这个 Fluent 收敛问题。

我提供的信息：
- 残差曲线：[截图/数据]
- 监控量：[力/压降/质量流量/温度等]
- Fluent warning/error：[粘贴日志]
- 网格质量：[skewness/orthogonal quality/y+]
- 边界条件：[入口/出口/壁面]
- 模型设置：[湍流/能量/多相/瞬态]

请判断：
1. 这是数值问题、物理设置问题、网格问题还是边界条件问题？
2. 当前结果是否能算收敛？
3. 最小修改方案是什么？
4. 修改后应该做什么 smoke test？
5. 最终应该用哪些指标验收？
```
